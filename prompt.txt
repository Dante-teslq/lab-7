Sessão 1 — Dataset

Crie o arquivo 01_generate_dataset.py em Python que:
- Use a API do Google Gemini com a biblioteca google-generativeai
- Leia a GEMINI_API_KEY via python-dotenv de um arquivo .env
- Modelo: gemini-1.5-flash
- Domínio: assistente especialista em gestão de estoque e WMS
- Gere 60 pares {"prompt": "...", "response": "..."} 
- Split: 90% → dataset/train.jsonl | 10% → dataset/test.jsonl
- Barra de progresso com tqdm
- Printar total de pares gerados ao final

Ao terminar, mostre o arquivo criado e confirme que roda sem erros de importação.


Sessão 2 — Fine-tuning
Crie o arquivo 02_finetune_qlora.py com o pipeline completo de fine-tuning. Requisitos exatos:

Quantização (bitsandbytes): load_in_4bit=True, bnb_4bit_quant_type="nf4", bnb_4bit_compute_dtype=torch.float16
LoRA (peft): task_type=CAUSAL_LM, r=64, lora_alpha=16, lora_dropout=0.1, bias="none", target_modules=["q_proj","v_proj"]
TrainingArguments: optim="paged_adamw_32bit", lr_scheduler_type="cosine", warmup_ratio=0.03, num_train_epochs=1, fp16=True
Modelo base: "NousResearch/Llama-2-7b-chat-hf"
Usar SFTTrainer da biblioteca trl
Salvar com trainer.model.save_pretrained("./lora-adapter") ao final
Aceitar --train_file e --test_file via argparse
Verificar GPU com fallback para CPU

Todas as configs devem estar explicitamente comentadas no código.
